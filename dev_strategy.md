# 주식 분석 웹사이트 개발 전략 📋

> 종목별 분석 대시보드를 체계적으로 확장하기 위한 로드맵

---

## 1. 현재 상태 & 목표

**보유 자산**: RKLB 대시보드 (production-grade) + 재사용 가능 템플릿 (MD/HTML)

**최종 목표**: 여러 종목의 분석 리포트를 하나의 웹사이트에서 탐색할 수 있는 포트폴리오 분석 허브

## 2. 아키텍처 옵션 비교

| 항목 | 내용 |

## 3. 추천 개발 로드맵

### Phase 1: 기반 구축 (1~2일)

**목표**: 현재 HTML 템플릿을 모듈화

```
project/
├── index.html              ← 종목 목록 홈페이지
├── template.html           ← 공통 대시보드 셸
├── css/
│   └── dashboard.css       ← 디자인 시스템 (현재 <style> 추출)
├── js/
│   ├── charts.js           ← Chart.js 렌더 엔진 (현재 <script> 추출)
│   └── render.js           ← DOM 렌더링 로직
├── data/
│   ├── RKLB.json           ← CONFIG 객체를 JSON으로 분리
│   ├── PLTR.json
│   └── TSLA.json
└── img/                    ← 필요 시 로고 등
```

**핵심 작업**:
- `<style>` → `dashboard.css` 추출
- `<script>` 렌더 엔진 → `render.js` 추출
- CONFIG 객체 → 종목별 `.json` 파일로 분리
- `template.html`이 URL 파라미터(`?stock=RKLB`)로 JSON을 로드하도록 수정

### Phase 2: 홈페이지 (1일)

**목표**: 종목 목록 + 네비게이션

홈페이지 기능:
- 포트폴리오 종목 카드 그리드 (종목명, 현재가, 변동률, 종합 점수)
- 섹터별 필터링
- 각 카드 클릭 → 해당 종목 대시보드로 이동
- 다크 테마 일관성 유지

### Phase 3: 종목 추가 자동화 (2~3일)

**목표**: Claude 프롬프트 한 번으로 새 종목 JSON 생성

워크플로우:
```
사용자: "PLTR 분석해줘"
  ↓
Claude: 웹 검색으로 최신 데이터 수집
  ↓
Claude: CONFIG 형식의 JSON 생성
  ↓
data/PLTR.json 저장
  ↓
홈페이지에 자동 등록
```

**필요 작업**:
- Claude 프롬프트 템플릿 작성 (JSON 출력 형식 지정)
- JSON 스키마 검증 스크립트
- `data/` 폴더 스캔 → 홈페이지 자동 갱신

### Phase 4: 비교 기능 (선택)

- 종목 간 비교 대시보드 (레이더 차트 오버레이)
- 포트폴리오 전체 섹터 배분 시각화
- 워치리스트 / 보유 종목 구분

## 4. 기술 스택 추천

| 레이어 | 추천 | 이유 |
|--------|------|------|
| 호스팅 | GitHub Pages / Vercel | 무료, 정적 사이트 최적 |
| CSS | 현재 디자인 시스템 유지 | 이미 프로덕션급 품질 |
| 차트 | Chart.js 4.x | 이미 구현됨, 가볍고 강력 |
| 데이터 | JSON 파일 | 프레임워크 없이 동적 로딩 가능 |
| 빌드 | 불필요 (Phase 1~3) | Vanilla JS로 충분 |
| 프레임워크 | 나중에 필요 시 Astro 고려 | 정적 사이트 + 부분 SPA |

## 5. 데이터 수집 전략

### 자동 수집 소스 (Claude 웹 검색)

| 데이터 | 소스 | 신뢰도 |
|--------|------|--------|
| 실적/재무 | SEC Edgar, 기업 IR | ★★★★★ |
| 주가/밸류에이션 | Yahoo Finance, MarketWatch | ★★★★☆ |
| 애널리스트 | MarketBeat, TipRanks | ★★★★☆ |
| 산업 전망 | Grand View Research, Statista | ★★★☆☆ |
| 경쟁사 | 직접 비교 분석 | ★★★☆☆ |

### Claude 프롬프트 예시

```
다음 종목을 분석하고, 아래 JSON 스키마에 맞춰 데이터를 생성해주세요.

종목: {TICKER}
기준일: {오늘 날짜}

요구사항:
1. 웹 검색으로 최신 실적 데이터 확인
2. CONFIG 객체의 모든 필드를 채울 것
3. 초보 투자자도 이해할 수 있는 설명 포함
4. 차트 데이터는 숫자 배열로 정확히 제공
5. 리스크와 강점은 균형 있게 5개씩

JSON 스키마:
[stock_analysis_template.html의 CONFIG 구조 붙여넣기]
```
